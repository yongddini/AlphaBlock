# WAN-100: 채택 진입 경로(B안 지정가)의 「첫 탭 RSI 무관 진입」 누락 — 수정 전/후 델타

**재현**

```
# 수정 후(현재 코드) 수치 — 아래 '후' 열
python -m backtest.wan95_zone_limit_report      # → wan95_zone_limit_recompute.csv

# 수정 전 수치 — 아래 '전' 열
#   `simulate_zone_limit_trade`의 first_tap_free 인자와
#   `build_zone_limit_candidates`의 전달부를 되돌린 뒤 같은 커맨드.
#   (커밋 04bf3c7 시점의 wan95_zone_limit_recompute.csv가 그 산출물이다)
```

델타 원본: `wan100_first_tap_free_delta.csv` (심볼 × TF, `zone_limit` 행만).

## 1. 누락은 사실인가 — **예**

CLAUDE.md·WAN-81이 정한 규칙은 「존 확정 후 **첫 탭은 RSI 무관 무조건 진입**(워밍업
NaN이어도), **재탭부터** RSI 게이트」다(`rsi_gate_mode="first_tap_free"`). 이 규칙이
**채택 기본 진입 경로인 B안(지정가)에서 전혀 적용되지 않았다.**

근거는 세 겹이다.

1. **게이트 함수는 첫 탭 면제를 스스로 하지 않는다.** `strategy/models.py`의
   `rsi_gate_passes`는 `mode="first_tap_free"`를 받아도 `extreme`과 같은 극단 규칙
   (롱 `RSI<=30`)으로 폴백한다. 의도된 설계다 — 이 함수는 `tap_index`를 모르므로
   **면제는 호출부 책임**이라는 계약이고, docstring에도 그렇게 적혀 있었다.
2. **A안은 그 계약을 지킨다.** `strategy/confluence.py`의 `_evaluate_entry`가
   `params.rsi_gate_mode == "first_tap_free" and signal.tap_index == 0`이면 무조건
   통과시킨다.
3. **B안은 지키지 않았다.** `backtest/substep.py`의 `simulate_zone_limit_trade`에는
   `tap_index` 개념 자체가 없고, 호출부 `backtest/zone_limit_backtest.py`에도 첫 탭
   분기가 없었다. 게다가 체결 조건이 `live_rsi is not None and rsi_gate_passes(...)`라
   **워밍업 구간에서는 첫 탭조차 진입이 막혔다** — 규칙의 "워밍업 NaN이어도 진입"과
   정확히 반대다.

수정 전 코드에 채택 기본값(`rsi_gate_mode="first_tap_free"`)을 그대로 넣고 첫 탭을
재현하면 둘 다 미체결(`no_touch`)로 나온다:

| 상황 | 규칙이 요구하는 동작 | 수정 전 실제 동작 |
| -- | -- | -- |
| 첫 탭 + 게이트 미충족 RSI(롱인데 과매수) | 무조건 진입 | **미체결** |
| 첫 탭 + RSI 워밍업(값 없음) | 무조건 진입 | **미체결** |

회귀 고정 테스트: `tests/test_substep.py`의 `test_first_tap_free_*` 3종 +
`test_retap_still_applies_rsi_gate_under_first_tap_free_mode`.

## 2. 무엇을 고쳤나 — 코드를 규칙에 맞췄다

규칙(CLAUDE.md)을 정본으로 두고 B안을 규칙에 맞췄다(사용자 확정). `tap_index`는 이미
시그널에 붙어 있었으므로(`EntryCandidateSignal.tap_index`) 배선만 하면 됐다.

- `backtest/substep.py`: `simulate_zone_limit_trade(first_tap_free: bool = False)` 추가.
  참이면 RSI 게이트를 건너뛰고 워밍업(None)이어도 지정가 터치 즉시 체결한다. 면제는
  "조건 통과"로 판정되므로 `cancel_on_condition_fail`의 취소 분기도 타지 않는다.
- `backtest/zone_limit_backtest.py`: 루프에서
  `first_tap_free = (effective_gate_mode == "first_tap_free" and signal.tap_index == 0)`을
  계산해 전달. 병합 존이면 `tap_index`가 이미 병합 존 기준으로 매겨져 있어 WAN-81 §5
  동작이 그대로 보존된다.
- `strategy/models.py`: `rsi_gate_passes` docstring의 "호출부가 처리한다" 계약이 **두
  경로 모두**의 책임임을 명시(누락이 조용히 지나간 이유가 이 계약의 암묵성이었다).

기본값은 바뀌지 않았다 — `first_tap_free=False`가 기본이라 이 인자를 넘기지 않는
호출부(WAN-70 무작위 대조군 등 `rsi_gate_mode` 오버라이드 경로)는 동작이 동일하다.

## 3. 수정 전/후 성과 델타 (3심볼 평균, 채택 기본값 + 펀딩 반영)

| TF | return 전 | return 후 | 델타 | 체결률 전 | 체결률 후 | 거래수 전 | 거래수 후 | 승률 전 | 승률 후 |
| -- | --: | --: | --: | --: | --: | --: | --: | --: | --: |
| 15m | 15.80% | **82.30%** | +66.50%p | 28.50% | 50.29% | 1383 | 2362 | 49.18% | 51.65% |
| 1h | 9.06% | **35.84%** | +26.77%p | 31.11% | 49.59% | 411 | 688 | 47.92% | 51.72% |
| 4h | 3.02% | **12.08%** | +9.06%p | 40.40% | 56.38% | 115 | 171 | 45.24% | 50.83% |
| 1d | −1.48% | **1.02%** | +2.50%p | 40.79% | 57.86% | 20 | 30 | 33.33% | 44.37% |

읽는 법: **체결률이 약 29% → 약 50%로 오른 것이 1차 효과**다. 첫 탭에 걸려 있던 극단
RSI 게이트(롱 `RSI<=30`)는 통과가 드물어 사실상 대부분의 첫 탭을 버리고 있었고, 그
버려진 첫 탭이 되살아나면서 거래 수가 1.5~1.7배로 늘었다. 승률도 함께 올랐다
(예: 15m 49.18% → 51.65%) — 첫 탭이 재탭보다 좋은 진입이라는 WAN-81의 전제와 방향이
일치한다.

전 열의 15m 평균 +15.80%·1h +9.06%는 CLAUDE.md와 WAN-96이 인용하던 바로 그 값이다
(각각 "+15.8%", "+9.1%") — 즉 이 표의 '전'은 기존 문서가 근거로 삼던 수치와 같은
지점이며, 비교가 성립한다.

### WAN-100과 무관한 차이 한 건 (기록)

재산출에서 `SOL 15m close` 행이 커밋본과 다르다(거래 939→938, return −8.68%→−9.87%).
**수정을 되돌리고 돌려도 똑같이 재현되므로 원인은 이 수정이 아니라 데이터 드리프트다**
— 펀딩비 조회가 `include_predicted=True`라 예측 펀딩률이 사후 확정되며 값이 갱신된다
(해당 행 펀딩 73.38→71.75). 나머지 11개 `close` 행은 비트 단위로 동일하다. `close`는
A안 경로라 애초에 첫 탭 면제가 적용돼 있었고, 이 11행의 불변이 그 사실의 대조 증거다.

## 4. WAN-96/99의 「15m 제외 · 1h 보류」 결론은 유지되는가

**답: 유지되지 않을 가능성이 높다. 다만 이 이슈에서 확정하지 않는다 — 재판정은 WAN-97
소관이다.**

WAN-96의 15m 제외 권고는 **최악 체결 가정**(`pen_5bp_drop_50` = 5bp 관통 요구 + 50%
탈락)에서 15m이 **−5.8%(0/3 플러스)** 로 무너진다는 데 걸려 있었다. 1h도 같은 가정에서
−1.7%(0/3)라 "보류"였다. 교정 엔진으로 그 축을 다시 돌리면 **두 수치의 부호가 뒤집힌다**:

| TF | 최악 가정 평균 return (교정 전) | (교정 후) | 플러스 심볼 전 → 후 |
| -- | --: | --: | --: |
| 15m | **−5.8%** | **+3.57%** | 0/3 → **3/3** |
| 1h | **−1.7%** | **+8.34%** | 0/3 → **3/3** |

즉 **제외·보류 판정의 근거였던 "최악 가정에서 마이너스"가 교정 후에는 성립하지 않는다.**
WAN-96의 결론이 틀렸다기보다, 그 결론이 **명세대로 돌지 않은 엔진 위에서 내려졌다**는
뜻이다.

**그러나 이것이 곧 "15m 채택"은 아니다.** 반대 신호가 남아 있다:

- **15m의 시드 분산이 크다.** 최악 가정 15m은 시드별로 −7.71% ~ +17.38%(BTC)로 널뛴다.
  평균 +3.57%는 **개별 시드가 마이너스인 경우를 포함한 값**이라, "최악 가정에서 견딘다"고
  말하기엔 근거가 얇다. 1h는 상대적으로 견고하다(ETH·SOL은 5시드 전부 플러스).
- **WAN-96의 진단 자체는 살아 있다.** `pen_5bp`(관통 요구)만 걸어도 15m은 +82.30% →
  +30.62%로 수익의 63%가 사라진다(거래 수는 787→739로 6%만 감소). 수익이 **지정가를
  스치듯 닿은 체결에 몰려 있다**는 비대칭은 교정 후에도 그대로다 — 다만 교정 전에는 그
  잠식이 수익을 **0 아래로** 끌어내렸고, 교정 후에는 플러스가 남는다는 차이다.
- 이 재산출에는 **오프셋 축·IS/OOS 분할이 빠져 있다**(WAN-99가 하던 과최적화 방어).

따라서 이 리포트의 역할은 **"WAN-96/99의 결론을 교정 엔진에서 다시 판정해야 한다"는
사실을 확정하는 것**까지다. 무엇을 실제로 돌릴지는 WAN-97이 전체 재산출로 결정한다.

주의: **WAN-96/99 표의 셀과 위 수치를 직접 비교하지 말 것.** 그 표들은 교정 전 엔진이며
(두 요약 md 헤더에 경고 명시), 특히 `wan99_zone_limit_offset.csv`는 회귀 테스트가 고정한
1셀(BTC 1h, full, offset 0, `pen_5bp`)만 교정 엔진으로 갱신돼 **엔진이 섞여 있다**.
