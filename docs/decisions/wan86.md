# WAN-86: WAN-84 "엣지 없음" 판정 이후의 갈림길

## 배경

[WAN-84](https://linear.app/wannabe-millionaire/issue/WAN-84/검증-신-엔진wan-81-oos워크포워드-무작위-진입-매칭-널-재검정-새-규칙에-엣지가-있는가)(PR #64,
재현: `uv run python -m backtest.wan84_new_engine_validation`, 리포트:
[`backtest/reports/wan84_summary.md`](../../backtest/reports/wan84_summary.md))는
신 엔진(WAN-81, `ConfluenceParams()` 기본값 — 볼린저 진입가·첫탭 무조건/재탭
RSI·고정 1:1.5R 익절·숏 활성화) 기준으로도 **통계적 엣지가 확인되지 않았다**고
결론지었다:

- **매칭 널**: 3심볼×4TF×IS/OOS 유효 셀 17개(거래 20건 이상) 중 p≤0.05 &
  실제수익>무작위평균인 셀 **0개**.
- **비용 민감도**: 다수 셀이 수수료·슬리피지 1.5x/2.0x 배율에서 음전환
  (예: BTC 1h OOS +5.0% → -4.1%).
- **롱/숏 분해**: OOS 기준 롱 평균 총수익률 **+3.07%**(유효 9셀) vs 숏 평균
  총수익률 **−3.82%**(유효 7셀) — 숏이 롱 수익을 갉아먹는다.

숏 활성화는 [WAN-81](https://linear.app/wannabe-millionaire/issue/WAN-81/전략엔진-메인-엔진-규칙-전면-교체-볼린저-진입가-첫탭-무조건재탭-rsi-고정-115-익절)에서
**사용자가 명시적으로 확정한 규칙**이므로 PM/개발자가 임의로 되돌리지 않는다.
반면 위 데이터는 숏이 롱 수익을 상쇄한다고 말한다. 이 충돌은 코드가 아니라
**문서화된 사용자 결정**으로 매듭짓는다.

이 문서가 다루는 결정 항목은 3개다. 아래 각 항목의 "결정" 칸은 **사용자 입력이
필요한 자리**이며, 이 PR을 연 시점에는 비워 둔다.

## 결정 항목 1 — 숏 유지 여부

**근거**([wan84_summary.md](../../backtest/reports/wan84_summary.md) §"롱/숏 분해"):
OOS 롱 평균 +3.07%(유효 9셀), 숏 평균 −3.82%(유효 7셀). 숏이 전체 성과를
끌어내린다.

선택지:

1. **규칙 우선** — WAN-81에서 확정한 대로 숏을 계속 활성화 유지(`short_enabled=True`).
2. **데이터 우선** — 숏을 비활성화(`short_enabled=False`)해 롱 온리로 되돌린다.
3. **분리** — `short_enabled`를 프로덕션 기본값과 별개로, 백테스트/파라미터 탐색용
   플래그로 분리해 두 경로를 모두 유지한다.

## 결정 항목 2 — 실거래 보류 여부

**근거**: "엣지 없음" 판정(매칭 널 유효 셀 0/17) 상태에서 실주문을 자동 활성화할
근거가 없다.

**결정**: 실거래는 계속 보류한다. `ALPHABLOCK_LIVE_TRADING` 기본값 `false`
유지(`config/settings.py:182`, `.env.example:182`). 엣지가 확인되기 전까지
사용자의 명시적 승인 없이는 이 값을 바꾸지 않는다.

**근거**: 매칭 널·비용 민감도·워크포워드(변형 B) 모두 엣지 신호가 약하거나 없음을
가리킨다([wan84_summary.md](../../backtest/reports/wan84_summary.md) §"결론").
