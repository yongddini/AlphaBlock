## 요약 — 결론 먼저

1. 📌 **§1 롱 축: 밴드를 바꿔도 판정이 안 움직인다.** 공식 렌즈 유효 셀 24개 중 유의
   **11개**로 WAN-124(`tap`)와 **같은 개수**이고, TF 분포도 같은 모양이다
   (**새 밴드 15m 8/12 · 1h 3/12** vs `tap` 15m 9/12 · 1h 2/12). WAN-132 §6이 남긴
   "다시 돌려야 답이 나온다"에 답하면: **답은 바뀌지 않았다.** §2.
2. 📌 **§2 부품 분해도 그대로다 — 볼린저가 유일한 플러스 부품이고 15m에서만이다.** 공식 렌즈
   OOS 볼린저 증분(`L1→L2`)이 **15m +16.70%p(6/6심볼) · 1h −2.80%p(2/6)**로, WAN-114의
   `tap` 판(15m +16.20%p · 1h −1.18%p)과 **부호가 같고 크기도 15m은 거의 같다**(1h는 더
   나빠졌다). §3.
3. 📌 **§3 숏 축(처음 재는 축): 판정 (c) — TF·구간에 갈린다.** `short_only` 유효 셀 24개 중
   유의 **8개**(15m 3/12 · 1h 5/12), `both`는 **13개**(15m 6/12 · 1h 7/12). 전 셀 일관이
   아니므로 **숏 재활성화의 채택 근거로는 약하다** — 이슈가 정한 (c)의 뜻 그대로다. §4.
4. 🚨 **그리고 숏의 OOS 플러스는 네 셀 전부 ETH 하나가 만든다** — `short_only` OOS
   15m +2.56% → ETH 제외 **−1.84%**, 1h +4.14% → **−0.52%**. `both`도 15m +11.53% →
   **−2.04%**, 1h +9.74% → **−1.92%**. WAN-89가 경고한 그 편중이 널 축에서도 그대로다. §4-2.
5. **기본값·토대는 바꾸지 않았다** — `short_enabled=False` 유지, 실거래 보류
   (`ALPHABLOCK_LIVE_TRADING=false`) 유지. **숏 재활성화 재-베이스라인은 제안하지 않는다**
   (판정이 (a)가 아니다). 사용자가 경고를 알고도 원하면 별도 결정 이슈다 — 개발자 임의 전환
   금지. §6.
6. ⚠️ **「엣지 찾았다」로 인용 금지** — 유의 셀은 `baseline`(낙관 렌즈, 「닿으면 체결」) 위의
   값이고, 이 널은 **「선별」과 「가격」을 가르지 못한다**(WAN-131 소관). §5.

## 0. 왜 다시 쟀나 — WAN-132가 스스로 남긴 구멍

WAN-132(PR #101)가 진입가 정본을 `band_bar="tap"` → `"intrabar_live"`로 옮기면서, 파급을
**재산출이 아니라 `tap` 명시 고정**으로 처리했다(28개 모듈). 그 선택은 옳다 — 결론 문장이
특정 수치에 박힌 리포트를 조용히 새 밴드로 돌리면 본문과 어긋난다. 하지만 대가로 **엣지 판정
계열 전체가 옛 엔진의 기록으로 얼어붙었다**:

* WAN-124 매칭 널(`wan123_matched_null`) — `tap` 고정(그 모듈의 상호 검산 48셀 차이 0과
  `wan111` CSV 비트 일치가 `tap`을 전제한다).
* WAN-114 부품 분해(`wan114_entry_rule_ablation`) — 역시 `tap` 고정.

⚠️ 하필 **두 판정의 주인공이 볼린저**다. 밴드 정의를 바꿔 놓고 볼린저의 기여·유의성을 옛
밴드에서만 재고 있었다는 뜻이다. 그래서 **옛 표는 손대지 않고 새 모듈 둘**을 냈다.

동시에 **숏 축을 처음으로 잰다**(구 WAN-148 흡수). WAN-89가 WAN-87의 숏 비활성화 근거를
무너뜨렸지만, 되살아난 숏 수익이 **무작위 진입과 구분되는지**는 한 번도 검정되지 않았다.
wan89.md §6-2: *"숏 축 매칭 널을 내기 전에는 「엣지」라는 단어를 쓰지 않는다."*

## 1. 자·축을 하나도 바꾸지 않았다 (그리고 그것이 이 표의 전부다)

밴드 말고 다른 것이 움직이면 "판정이 바뀌었다"와 "자를 바꿨다"를 구분할 수 없다. 그래서:

| 축 | WAN-124 / WAN-114 | 이 이슈 |
| -- | -- | -- |
| 밴드 | `tap`(고정) | **`intrabar_live`(채택 기본값 — 고정하지 않는다)** |
| 무력화 축(널) | 볼린저 | 볼린저(같음) |
| 자 | 거래 20건 이상 · p≤0.05 **&** 실제>무작위평균 | 같음 |
| 부트스트랩 | 200회 · 시드 124 | 같음 |
| 창·심볼·TF·구간 | 못 박은 창 · 6심볼 · 15m·1h · IS/OOS | 같음 |
| 렌즈 | `baseline`(+`pen_5bp`) / 3렌즈 | **`baseline` 단독**(WAN-128) |
| 팔 | 롱 온리 | + `short_only` · `both`(**WAN-89 정의를 import**) |

📌 **검산 세 개가 전부 차이 0이다** — 새 수치가 엔진을 잘못 태워 나온 것이 아님을 보증한다:

1. **§1·§3의 실제 팔 72행 ≡ `wan89_short_autopsy.csv`** (수익·거래 수·`buy_hold` 전부
   차이 `0.00e+00`). 즉 이 널이 검정하는 「실제」는 WAN-89가 이미 낸 그 성적표다.
2. **§2의 `L1`·`L2` 48행 ≡ `wan119_intrabar_live_band.csv`의 `L1`·`L2i`**(차이 0). WAN-119가
   옵트인으로 잰 봉내 라이브 밴드 단이 지금은 **기본값 경로**로 같은 값을 낸다.
3. **§2의 `L2u`(채택 기본값 단) 24행 ≡ §1의 `long_only` 실제 24행**(차이 0) — **두 새 모듈이
   서로를 검산한다.**

장세 라벨도 새로 정의하지 않았다 — WAN-89의 `_buy_hold`를 **같은 함수로 import**해 같은
행에 싣는다(IS 바이앤홀드 +255~265% = 상승장 · OOS −35% = 하락장).

## 2. §1 롱 축 — 판정은 밴드에 기대고 있지 않았다

공식 렌즈(`baseline`) 유효 셀(거래 20건 이상)의 유의 셀 수:

| TF · 구간 | `tap`(WAN-124) | **새 밴드(이 표)** |
| -- | --: | --: |
| 15m IS | 5/6 | **5/6** |
| 15m OOS | 4/6 | **3/6** |
| 1h IS | 1/6 | **2/6** |
| 1h OOS | 1/6 | **1/6** |
| **합** | **11/24** (15m 9 · 1h 2) | **11/24** (15m 8 · 1h 3) |

**총 개수가 같고 「15m ≫ 1h」라는 모양도 같다.** WAN-124가 낸 판정(볼린저 축에서 15m은
무작위와 구분되고 1h는 아니다)은 **밴드 정의의 산물이 아니었다.**

수익 수준은 조금 내려간다(심볼평균 실제 수익, `long_only`):

| TF · 구간 | `tap` | 새 밴드 |
| -- | --: | --: |
| 15m IS | +24.50% | +20.27% |
| 15m OOS | +8.63% | +5.64% |
| 1h IS | +17.91% | +17.81% |
| 1h OOS | +5.28% | +2.98% |

⚠️ **이 델타를 「전환 비용」으로 단독 인용하지 말 것** — WAN-132가 이미 낸 전환 비용
(15m OOS +0.50%p · 1h OOS −1.62%p)은 **게이트가 켜진 사다리 단(`L2`)** 기준이고, 위 표는
게이트가 없는 오늘의 기본값이다. 방향만 읽는다.

## 3. §2 부품 분해 — 볼린저가 유일한 플러스 부품이라는 결론도 그대로다

공식 렌즈 OOS 증분(심볼별로 짝지어 뺀 뒤 평균):

| 단 | 새로 켜는 부품 | 15m OOS | 1h OOS |
| -- | -- | --: | --: |
| `L0→L0r` | 재탭 노출 | −3.85%p (0/6) | +2.19%p (3/6) |
| `L0r→L1` | 재탭 RSI 게이트 | −2.76%p (3/6) | −2.28%p (2/6) |
| `L1→L2` | **볼린저** | **+16.70%p (6/6)** | **−2.80%p (2/6)** |
| `L2→L2u` | RSI 게이트 제거(= 오늘의 기본값) | +1.23%p (3/6) | +3.38%p (5/6) |

* **볼린저**: WAN-114의 `tap` 판은 15m OOS **+16.20%p** · 1h OOS **−1.18%p**였다. 새 밴드에서
  15m은 **+16.70%p로 사실상 같고**(6/6심볼 상승) 1h는 **−2.80%p로 더 나빠진다**. 부호가 두
  TF 모두 유지된다 = **결론 유지**.
* **RSI 게이트**(`L0r→L1`)는 **두 TF·두 구간 모두 마이너스**다(15m IS만 +2.78%p인데 OOS에서
  −2.76%p로 뒤집힌다 — WAN-114가 관찰한 IS→OOS 뒤집힘의 재현). **WAN-123의 게이트 제거
  결정이 새 밴드에서도 지지된다**: `L2→L2u`가 **네 셀 중 셋에서 플러스**(1h IS +3.99%p ·
  1h OOS +3.38%p · 15m OOS +1.23%p, 15m IS만 −1.53%p).
* **1h는 여전히 규칙 층 전체가 값을 못 더한다** — 존-단독 `L0`(+2.48%)와 채택 기본값
  `L2u`(+2.98%)의 격차가 **+0.49%p**뿐이고 IS에서는 **−0.14%p로 존-단독이 이긴다**.
* 📌 **체결률이 부품 층의 산물이라는 WAN-114 관찰도 재현된다** — 존-단독 100% → 볼린저가
  `L2`에서 48~49%로 깎고, 게이트 제거(`L2u`)가 **72~78%로 되돌린다**.

🚨 **표의 `L2`는 채택 기본값이 아니다 — 이 표에서 채택 기본값은 `L2u`다.** WAN-123이 게이트를
뺀 뒤 WAN-114 사다리의 `L2` 라벨("= 채택 기본값")은 거짓이 됐다. 설정은 그대로 물려받되
(그래야 두 표가 같은 것을 가리킨다) **표에 찍히는 문장만** 교정했고, 회귀 테스트가 그 교정과
「`L2u` == `ConfluenceParams()`」를 동작으로 고정한다.

## 4. §3 숏 축 — 처음 재는 축, 판정 (c)

### 4-1. 유의 셀

| 팔 | 15m IS | 15m OOS | 1h IS | 1h OOS | 합 |
| -- | --: | --: | --: | --: | --: |
| `long_only`(대조) | 5/6 | 3/6 | 2/6 | 1/6 | 11/24 |
| `short_only` | 2/6 | 1/6 | 4/6 | 1/6 | **8/24** |
| `both` | 3/6 | 3/6 | 5/6 | 2/6 | **13/24** |

**판정 (c) — TF·구간에 갈린다.** 숏은 롱 축의 「15m ≫ 1h」 패턴을 **따르지 않는다**: 숏의
유의성은 오히려 **1h IS(상승장)** 에 몰린다(4/6·5/6). 이슈가 (c)에 정해 둔 읽는 법 그대로,
**튜닝 대상이라 채택 근거로 약하다**.

### 4-2. 🚨 OOS 플러스는 ETH 하나가 만든다

| TF · 구간 · 팔 | 심볼평균 | ETH 제외 |
| -- | --: | --: |
| 15m OOS `short_only` | +2.56% | **−1.84%** |
| 1h OOS `short_only` | +4.14% | **−0.52%** |
| 15m OOS `both` | +11.53% | **−2.04%** |
| 1h OOS `both` | +9.74% | **−1.92%** |
| (대조) 15m OOS `long_only` | +5.64% | **−1.88%** |
| (대조) 1h OOS `long_only` | +2.98% | **−3.80%** |

**여섯 셀 전부 ETH를 빼면 마이너스**다. WAN-89가 낸 경고가 널 축에서도 그대로이고, **롱도
같다** — 즉 이것은 숏의 결함이 아니라 이 엔진 전체의 성질이다.

### 4-3. 장세 의존은 사실이다

15m 숏은 상승장 IS **−12.70%** / 하락장 OOS **+2.56%** 로 부호가 갈리고, 1h 숏은 두 구간 다
플러스(IS +17.90% · OOS +4.14%)다. **하나의 팔이 두 장세·두 TF에서 일관되게 무작위를 이기지
못한다**는 것이 (c) 판정의 내용이다.

## 5. 이 표가 말하지 않는 것

* ⚠️ **「엣지 찾았다」가 아니다.** 유의 셀은 전부 `baseline`(낙관 렌즈, 「닿으면 체결」·큐
  우선순위 미모델링) 위의 값이다. 실제 해소는 틱·호가(WAN-98) 소관인데 **그건 Canceled**다.
* ⚠️ **「선별」과 「가격」을 가르지 못한다** — 널의 풀은 존 근단 가격이고 실제는 밴드가다.
  WAN-96/114/115/120이 전부 **가격** 쪽을 가리키므로 유의성은 **「가격 효과 재확인」으로 먼저
  읽는다**. 가르는 것은 **WAN-131** 소관이고, 이 표가 그 이슈의 **새 밴드 기준선**이다.
* ⚠️ **숏 축 판정은 롱 축 판정(WAN-84/88/111/114/124)의 반박이 아니다** — 숏은 처음 묻는
  질문이다. 「엣지 없음」 계열은 그대로다.
* ⚠️ **널이 퇴화하지 않았다는 것은 산출물에도 보인다** — 무력화 풀이 실제 거래 수의 평균
  **3.04배**(최소 2.07배). 코드가 막고 있고(같은 파라미터면 `ValueError`) **숏 경로에서도
  막힌다는 것을 회귀 테스트가 동작으로 고정한다**(§3 완료기준).

## 6. 결정 · 다음

* **기본값·토대 불변** — `ConfluenceParams()` 그대로, `short_enabled=False` 유지, 실거래 보류
  (`ALPHABLOCK_LIVE_TRADING=false`) 유지. 이 이슈는 **검증 전용**이다.
* **숏 재활성화 재-베이스라인은 제안하지 않는다** — 판정이 (a)가 아니라 (c)이고, OOS 플러스는

  숏 재검 · 숏 손절 위치(wan89.md §6).
* **옛 `tap` 고정 리포트·CSV는 손대지 않았다** — 비트 재현 보존. 새 판정은 새 모듈로만 낸다.
* **다음**: WAN-131(볼린저의 선별 대 가격) — 이 표가 그 기준선이다.
