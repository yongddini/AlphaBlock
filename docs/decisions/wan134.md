## 요약 — 결론 먼저: (c) 두 TF 모두 병합은 손절률에 중립 · 남는 작은 차이는 존폭이 설명한다

사용자 질문 *"combined된 오더블록에서 더 손절이 많고 이런 게 있어?"* 에 대한 답:
**약한 경향은 보이지만 무작위와 구분되지 않고, 그 차이마저 병합이 아니라 존폭에서 온다.**

| TF | OOS 병합 손절률 | OOS 단일 손절률 | 손절차 | corr(combined,뚫림) | 심볼층화 순열 p |
| -- | -- | -- | -- | -- | -- |
| **15m** | 53.8% | 49.9% | +3.9%p | +0.034 | **0.160** (미달) |
| **1h** | 54.1% | 47.0% | +7.1%p | +0.058 | **0.187** (미달) |

1. **원 대조부터 매칭 널을 못 넘는다.** OOS에 병합존이 몇 %p 더 뚫리는 것처럼 보이지만(15m
   +3.9%p · 1h +7.1%p) 심볼 층화 라벨 순열에서 **p=0.16 · 0.19로 α=0.05 미달**이다. IS는 더
   약하고 방향도 흔들린다(15m +1.3%p p=0.59 · **1h −1.2%p p=0.79로 부호 반대**). 즉 OOS의
   차이는 무작위와 구분되지 않고 IS→OOS로 재현되지도 않는다 → **(c)**.
2. **그 작은 차이마저 존폭이 만든다(폭 통제 후 소멸 · WAN-117 축과 동일).** 병합존은 정의상
   넓다 — 평균 존폭 `zone_width_atr`이 **병합 2.8~3.0 vs 단일 1.6~1.7**. 존폭 분위로 통제하면
   통제 corr이 **+0.010~+0.058(p=0.46~0.99)로 유의성이 없다**. 분위별 손절률을 보면 **병합이든
   단일이든 넓은 분위(Q5)에서 나란히 더 뚫린다**(15m OOS 단일 Q1 37%→Q5 53% · 병합 Q1 39%→Q5
   61%). 뚫림을 가르는 건 병합 여부가 아니라 **존폭**이고, 이는 WAN-117이 뚫림의 유일 강건
   축으로 지목한 `zone_width_atr`(그마저 「고정 R 장벽 기하」로 의심)과 **정확히 같은 축**이다.
   → 형식 판정은 (c)이되 그 안의 미미한 차이는 **(b) 「병합은 존폭의 대리변수」**로 읽힌다.
   후속은 **WAN-133**(존폭 기하 대 선별)에 합류한다.
3. **심볼 편중 아님(단, 애초에 유의하지 않다).** leave-one-out 손절차가 어느 심볼을 빼도 같은
   부호의 작은 양수다(15m +1.2~+5.7%p · 1h +4.6~+9.1%p). 한 심볼이 만든 착시는 아니지만,
   풀 자체가 유의하지 않으므로 "편중이 아니라서 유의하다"로 읽으면 안 된다.

**= WAN-84/88/111/114/124/117의 「좋은 존과 나쁜 존을 가르는 축이 존폭뿐이고 그마저 기하로
의심된다」를 병합 축에서 재확인.** 병합이 손절을 늘리는 독립적 드래그라는 증거는 없다.
채택(재-베이스라인) 권고 없음 — 기본값 불변(`combine_obs=True` 유지), 실거래 보류 유지.

## 두 개의 부수 발견 (완료 기준 4·3)

### (4) 거래량 오염은 실재한다 — 강하게

`_make_merged_group`이 병합존의 `ob_volume`을 **구성 존 합산**으로 만들어, 병합존의 거래량
퍼센타일이 기계적으로 부풀려진다. 실측:

| TF·구간 | 병합 vol% | 단일 vol% | corr(combined, volume_pctl) |
| -- | -- | -- | -- |
| 15m OOS | 69.5% | 43.2% | **+0.394** |
| 1h OOS | 70.6% | 39.8% | **+0.439** |

네 셀 전부 corr +0.39~+0.44다. **병합 여부가 곧 거래량 퍼센타일 상위**라는 뜻으로, 이는
WAN-77·WAN-117에서 거래량 축(`volume_pctl`)이 IS에서 유의했다가 **OOS에서 corr≈0으로
붕괴**한 것과 무관하지 않다 — 병합존의 합산 스케일이 거래량 특징에 병합 상태를 섞어 넣어
노이즈를 주입한다. **거래량 특징을 쓰는 후속 이슈는 병합존의 합산 스케일을 정규화하거나
단일 존으로 한정해 재검해야 한다**(기록용 · 이 이슈에서 고치지 않음 — 측정만).

### (3) Ablation: `combine_obs=False`는 손익을 뚜렷이 바꾸지 못한다 — 게다가 전부 거래 오염

`combine_obs`는 **탐지 시점** 파라미터다(`OrderBlockDetector.run`이 병합 여부에 따라
`retap_signals`를 다르게 낸다 — `order_blocks.py:566`). 그래서 각 팔은 자기 `combine_obs`로
**따로 탐지**한다(한 번 탐지해 공유하면 두 팔이 비트 단위로 같아지는 조용한 실패 — 회귀
테스트가 동작으로 막는다). 심볼평균 델타(off − on):

| TF·구간 | ret on→off | Δret | 거래 on→off | 거래 오염 |
| -- | -- | -- | -- | -- |
| 15m IS | +25.92→+27.24% | +1.32%p | 586→618 | **6% ⚠️** |
| 15m OOS | +8.72→+10.58% | +1.86%p | 280→301 | **8% ⚠️** |
| 1h IS | +18.53→+23.36% | +4.83%p | 167→179 | **7% ⚠️** |
| 1h OOS | +5.22→+4.19% | **−1.03%p** | 84→89 | **5% ⚠️** |

- **네 셀 모두 거래 수 오염 게이트(>5%)에 걸린다** — 병합을 끄면 겹치는 존들이 각자 진입을
  세어 거래가 5~8% 는다(WAN-56이 "병합 단위당 1회 진입"으로 접어 둔 것을 푸는 것). **거래
  집합 자체가 바뀌므로 델타를 순수 병합 효과로 인용하지 않는다**(WAN-123 게이트 제거·WAN-126
  겹침과 같은 부류).
- **방향도 일관되지 않는다** — 15m은 off가 조금 낫고(IS·OOS 둘 다 +) 1h는 갈린다(IS +4.83%p,
  **OOS −1.03%p로 반전**). IS→OOS가 어긋나므로 어느 쪽도 단독 인용 금지. MDD도 ±1%p 안에서
  움직여 결정적이지 않다.
- 즉 **손익 관점에서도 병합을 끌 근거가 없다**(있었다면 손절 드래그 가설이 살아났을 텐데,
  손절률(위 (c))·손익(여기) 어느 쪽에서도 병합이 순손해라는 신호가 없다).

## 방법 (WAN-117 재사용)

- **라벨링**: 채택 기본값으로 `build_zone_limit_candidates` → `sequence_with_candidates`(프로덕션
  동시 1포지션 시퀀서)로 거래를 내고, `_Candidate.reason`으로 뚫림(무효화 손절, R=−1)·버팀
  (1.5R 익절, R=+1.5)을 라벨링, `END_OF_DATA`는 제외. WAN-117 `_FeatureExtractor`로
  `zone_width_atr`를 같은 계산(직전 확정봉 ATR)으로 실어 두 부검이 교차 검산된다. 병합 축은
  `OrderBlock.combined`(+ 이번에 추가한 측정 필드 `num_component_obs`)에서 읽는다.
- **유의성**: 통계량 = corr(combined, 뚫림) 점이연 상관, 널 = **심볼 층화 라벨 순열**(각 심볼의
  무효화율·표본 보존, 특징↔결과 연관만 끊음) 2000회 양측. 단일 특징이라 Bonferroni 보정 없이
  α=0.05, **OOS 유의 & IS 동일 부호**를 요구한다(IS→OOS 부호 뒤집힘 방지).
- **폭 통제**: 순열의 층을 (심볼 × 존폭 분위 5분위)로 잡아 폭·심볼 구성을 보존한 채 병합↔뚫림
  연관만 끊는다. 폭을 통제해도 유의하면 (a), 사라지면 (b) 대리변수, 원 대조부터 유의하지
  않으면 (c).
- **Ablation**: on(`combine_obs=True`)·off(`False`)를 각자 탐지해 IS/OOS로 돌리고,
  total_return·MDD·거래수·체결률을 대조 + 거래 수 오염 게이트.

## 엔진 변경 (측정용, 비파괴)

- `OrderBlock.num_component_obs: int = 1` 추가 — 병합에 참여한 구성 존 개수. `_make_merged_group`
  병합 분기에서 `len(obs)`로 채우고 단일 존·기존 생성 경로는 기본값 1이라 진입·손절·익절
  로직·기존 리포트는 비트 단위로 재현된다(측정 주석일 뿐). WAN-90이 `mfe_r`/`mae_r`를 잰
  것과 같은 성격이다.
