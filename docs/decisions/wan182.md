# WAN-182: 채택 유니버스 9종목 · 창 6년 · 4h 작업 TF 승격 — WAN-179 결정 실행

**성격: 재-베이스라인(실행).** WAN-179에서 사용자가 확정한 결정(2026-07-23) 셋을 코드
기본값·문서·리포트 파급에 반영한다. **새 격자·새 수치는 없다** — 근거는 WAN-176(전략
판정, PR #129) · WAN-180(레버리지, PR #130)이 이미 낸 표이고, 이 이슈는 배선·핀·문서다.

1. **채택 유니버스 6 → 9종목**: 기존 BTC·ETH·SOL·BNB·XRP·TRX + **DOGE·LINK·LTC**.
2. **채택 창 3년 → 6년**: 못 박은 창 **2020-09-15 ~ 2026-07-22**(실효 5.85년).
   시작점은 SOL 상장(2020-09-14) 하한 — 9종목이 같은 창을 보는 균일 창 원칙(WAN-175/176).
   끝은 마지막 미완 하루를 잘라낸 2026-07-22 00:00 UTC.
3. **4h 작업 TF 승격**: 앞으로 측정은 **15m·1h·4h 세 축**. 1d는 표본 미달로 제외 유지.

⚠️ **이번에 움직인 것은 엔진이 아니라 좌표다** — `ConfluenceParams()`(오프셋 2bp ·
`intrabar_live` · `unconditional` · 고정 1.5R · 롱 온리 · `combine_obs=False` · 존폭 필터
1.28)는 그대로다. 따라서 옛/새 리포트의 차이를 엔진 변화로 읽으면 안 된다(창·유니버스가
통째로 다르다 — 셀 비교 금지).

## §1 기본값 이동 — 무엇이 어디에서 바뀌었나

이슈가 물은 「6년 창·세 축이 코드 기본값인가 CLAUDE.md 관례인가」의 답: **관례였다.**
harness의 기본 좌표는 WAN-101 이래 3심볼 × 1h × `--years 3`(미끄러지는 창)이었고, 못 박은
창·6심볼은 각 리포트 모듈이 자기 상수로 들고 있었다. 이번에 채택 좌표를 **코드 기본값**으로
올렸다:

| 위치 | 옛 값 | 새 값 |
| -- | -- | -- |
| `config.settings._default_symbols` (수집) | 6심볼 | **9종목** |
| `backtest/harness.py DEFAULT_SYMBOLS` | 3심볼 | **9종목** |
| `harness.DEFAULT_TIMEFRAMES` | `("1h",)` | **`("15m", "1h", "4h")`** |
| `harness.DEFAULT_START/END` (신설) | — | **`2020-09-15` / `2026-07-22`** |
| `backtest.run` 인자 없는 실행의 창 | `--years 3`(미끄러짐) | **못 박은 채택 창** |
| `backtest.wan95_zone_limit_report` 좌표 | 3심볼 × 4TF × 3년 | **9종목 × 3TF × 채택 창** |

- **`--years N`을 명시하면 옛 미끄러지는 창 그대로다** — 채택 창은 창 인자가 하나도 없는
  실행에만 적용된다(`options_from_args`가 유일한 결정 지점). `--start`/`--end` 명시가 이긴다.
- **정본 리포트 = `uv run python -m backtest.run --oos-warm`**(인자 없이) — 이제 9종목 ×
  15m·1h·4h × 채택 창에서 `oos_warm`(주) + `oos`(스트레스)를 병기한다(WAN-166).
- **실거래/시그널 대상은 넓히지 않았다** — `live_signal_symbols` 기본 BTC 단독 유지,
  `ALPHABLOCK_LIVE_TRADING=false` 유지. 유니버스 확장은 **측정·수집 대상**이지 실거래
  승인이 아니다(WAN-111 원칙).

## §2 옛 리포트 파급 — 핀 · 무변 · 재산출

`combine_obs`(WAN-149)·`band_bar`(WAN-132) 때와 달리 파급이 작다: **옛 리포트 대부분이
좌표를 자기 모듈 상수로 들고 있어**(3심볼/3년을 스스로 정의) harness 기본값 이동의 영향을
받지 않는다. harness 기본값을 **직접 읽던** 모듈만 핀이 필요했다.

| 분류 | 모듈 | 처리 | 이유 |
| -- | -- | -- | -- |
| **핀(`LEGACY_*`)** | `wan104_offset_incremental_report` | `DEFAULT_SYMBOLS/YEARS = LEGACY_*` | 0bp vs 2bp 판정이 3심볼/3년 수치에 박혀 있다. harness에서 직접 임포트하던 유일한 심볼 소비자 |
| **핀(`LEGACY_*`)** | `wan108_multi_position_reappraisal` | `DEFAULT_YEARS = LEGACY_YEARS` | 다중 대 단일 판정이 3심볼/3년 수치에 박혀 있다 |
| **무변(자기 좌표)** | wan68/70/71/73/74/75/76/77/81/83/84/87/88/91/96/99 · ab_experiment · ab_walkforward | 없음 | 3심볼/3년을 **자기 모듈 상수**로 정의 — 기본값 이동과 무관하게 같은 행을 낸다 |
| **무변(못 박은 창)** | wan88(창 부분)/89/90/110~176 계열 | 없음 | 창·유니버스를 자기 입력으로 명시(`2023-07-14~2026-07-15` 등 못 박은 창) |
| **재산출(추적)** | `wan95_zone_limit_report` | 새 좌표로 재배선 + 전면 재산출 | "지금 채택된 것"을 재는 리포트는 고정하지 않는다(WAN-132/149/159와 같은 원칙) |

- `harness.LEGACY_SYMBOLS`(3심볼)·`LEGACY_TIMEFRAMES`(`("1h",)`)·`LEGACY_YEARS`(3.0)를
  신설했다 — 값이 옛 기본값과 동일하므로 핀된 모듈은 비트 단위로 재현된다.
- ⚠️ **옛 wan95 md/csv와 새 판을 셀 비교하지 말 것** — 좌표가 통째로 다르다. 옛 판
  (3심볼 × 4TF × `--years 3`)은 git 이력에 남는다.
- 창이 못 박혀 있으므로 **wan95는 이제 새 봉이 쌓여도 비트 단위로 재현된다** — 옛 판의
  「데이터 드리프트」 각주(`--years 3` 창 미끄러짐) 원인 하나가 사라졌다.
- **재산출 헤드라인**(지정가 팔, 심볼평균 · 전 구간): 15m **+355.5%**(9/9 플러스, MDD 8.50%,
  체결률 72.7%) · 1h **+107.9%**(9/9, MDD 7.23%, 68.7%) · 4h **+31.2%**(9/9, MDD 4.18%,
  68.2%). 🚨 6년 복리의 전 구간 절대값이라 **인용은 방향·대조로만**(WAN-169/180 경고와 같은
  부류) — `baseline` 낙관 렌즈 위의 값이고 IS/OOS 분해가 아니다(그쪽은 WAN-176 표가 정본).
- **실행 각주**: 9종목 × 6년 직렬 재산출은 단일 프로세스에서 메모리 피크로 조용히 죽었다
  (1차 시도, 트레이스백 없이 프로세스 소멸). `collect_rows`에 심볼 간 1분봉 해제(`del` +
  `gc.collect()`)를 넣었고, 실제 산출은 심볼별 서브프로세스로 격리해 냈다 — 행 단위 병합
  결과이며 렌더는 wan95 자체 함수를 그대로 썼다.

## §3 신규 3종목 펀딩 공백 — WAN-178 관계

DOGE·LINK·LTC는 이 창에서 `funding_rate` 테이블이 **0행**이다(실측: 기존 6종목만
5,545~5,633행). 그대로 두면 펀딩비 0으로 성과가 부풀려진다.

- **채택 성과(wan95)는 WAN-180과 같은 대리 규칙으로 보정한다**(사용자 지시 2026-07-23):
  대리 종목 = 기존 종목 중 확정 펀딩 평균 **최고**(= 롱에게 가장 비싼) 종목의 시계열.
  보수적 대체다. `apply_funding_proxy`는 **자기 데이터가 있으면 덮지 않는다** — WAN-178
  백필이 끝나면 저절로 아무것도 하지 않는다(재산출하면 실데이터로 돈다).
- **범용 CLI(`backtest.run`)는 대리를 얹지 않는다** — 신규 3종목 행은 펀딩 미반영
  (`funding_coverage` 0%)으로 나온다. 표를 읽을 때 이 열을 확인할 것.
- 수집기는 settings 확장으로 앞으로의 펀딩을 쌓지만 **과거분은 WAN-178(백필) 소관**이다.
  백필 후 wan95 재산출로 대리를 실데이터로 교체하는 재확인이 필요하다.

## §4 엣지 판정 계열의 동결 — 새 창 재검은 별도 이슈

**WAN-84/88/111/114/124/151(엣지·부품·널 판정)은 6종목/3년(및 3심볼/3년) 기록으로
얼어붙는다.** WAN-132→WAN-145, WAN-149→WAN-151과 같은 패턴이다 — 옛 표는 손대지 않고,
새 베이스라인 위의 재검은 별도 이슈로 낸다.

- **WAN-176이 이미 새 창(9종목·6년)에서 매칭 널·존폭 매칭·TF·다중/단일을 냈다**
  (`backtest/reports/wan176_summary.md`) — 새 창 판정의 정본 근거는 그 표다.
  ⚠️ **6종목/3년 판정과 한 표에 섞지 말 것**(창·유니버스가 다르다).
- **스핀아웃 권고**: WAN-176이 안 낸 축(부품 분해 ablation(WAN-114/145/151 계열의 9종목·
  6년 판) · 숏 축 널 · 체결 보수화 축)의 새 창 재검이 필요해지면 별도 이슈로 연다 —
  개발자 임의 착수 금지.
- 🚨 **「엣지 없음」은 뒤집힌 게 아니다** — 유니버스·창을 넓혀 재측정했을 뿐이다(WAN-176).
  전부 `baseline` 렌즈 위의 값이고 체결 보수화는 미측정이다.

## §5 회귀 테스트 (`tests/test_wan182_rebaseline.py`)

라벨이 아니라 **동작**으로 고정한다:

- 인자 없는 CLI가 실제로 9종목 × 15m·1h·4h × 못 박은 채택 창을 돈다(파서 산출물 검증).
- `--years` 명시는 미끄러지는 창 유지 · `--start`/`--end` 명시가 채택 창을 이긴다.
- 수집 유니버스 9종목 · 실시간 시그널 BTC 단독 불변.
- wan104/wan108은 `LEGACY_*` 좌표 · wan81 등 자기 좌표 모듈은 3심볼/3년 그대로.
- wan95 기본 인자는 harness 채택 좌표를 따른다(시그니처 검증).
- 펀딩 대리: 빈 신규 종목만 채움 · 자기 데이터 보존 · 백필 후 자동 무동작 · 확정값 없는
  종목은 대리 후보 불가.
- 기존 회귀(`tests/test_run_regression_real_data.py`)는 창·심볼을 명시 인자로 못 박아
  돌므로 이 전환의 영향을 받지 않는다(옛 엔진·옛 좌표 대조가 그대로 성립).
